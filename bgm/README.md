# BGM 素材放這裡(Phase E)

遊戲在**執行時**用相對路徑 `bgm/<id>.ogg` 抓 BGM(不進 PWA precache 離線包;
首次線上抓、之後由 service worker 的 CacheFirst 規則離線供應——見 `vite.config.ts`)。

## 需要的四個檔(檔名要完全一致)

| 檔名 | 場景 | 基調(kickoff §3) |
|------|------|------------------|
| `city.ogg` | 灰燼堡城內 | 平靜/日常、安穩 |
| `wasteland.ogg` | 灰燼荒原 | 蒼涼/危險、開闊不安 |
| `cave.ogg` | 東邊地穴 | 幽閉壓迫(與荒原開闊刻意對比) |
| `boss.ogg` | Boss 戰 | 張力 |

## 格式

- 預設副檔名 `.ogg`(Vorbis)——**無縫循環最佳**、體積小、Android 與 iOS 17+ 支援。
- 若你的手機是**較舊的 iOS**(不支援 ogg),改用 `.mp3` 或 `.m4a`,並告訴我把
  `src/game/bgm.ts` 的 `BGM_EXT` 常數改成對應副檔名(一行改動)。
- 檔案要**能無縫循環**(loop point 對齊):`source.loop = true` 會整段循環,
  素材本身首尾要接得上,否則每輪會有一下明顯的接縫。

## 容錯

- 檔案不存在/抓取失敗一律**靜默降級**——遊戲照常運作,只是該場景沒有配樂,
  永遠不會因為缺 BGM 而卡住場景載入或戰鬥(kickoff §2 容錯哲學)。
- 所以你可以一次只放一兩軌測試,不必四軌到齊。

## 授權

放 **CC0 / 公有領域**素材(或你有權使用的音檔)。建議把來源與授權附在本資料夾
(例如 `city.ogg` 的來源網址),方便日後查核。
